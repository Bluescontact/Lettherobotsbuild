# VCI (Values-Contribution Index) — Succession Assessment

## Status: Deferred to Mid-Succession

The VCI has been identified as a gap in two independent source analyses (The Missing Piece and Archive Pattern Analysis). This document records the explicit decision to defer specification rather than leave the gap unnamed.

## What the VCI Would Be

A mechanism for making visible how values align with contributions within a recognition-based system. Not a reputation score (reputation scores are legibility mechanisms that destroy local knowledge — see the Scale Without Legibility thread). Something more like: a way for the system to see whether what people say they value matches what they actually build.

This is the Differentiation vertex applied to participation: does behavior match claims?

## Why Defer

### Succession stage mismatch
The repository is at pioneer/early succession. VCI is mid-succession infrastructure — it requires:
- Enough participants to generate observable patterns (pioneer has few)
- Established exchange patterns to measure against (early succession is still forming these)
- Feedback loops for self-correction (mid-succession work)

Specifying VCI now would be premature formalization — doing mid-succession work at pioneer stage.

### The gatekeeping risk
Any index that measures contribution risks becoming a gatekeeping mechanism (see Entry Without Gatekeeping thread). The VCI would need to be designed *after* the entry sequence is operational and the graduated trust model has been tested. Designing the measurement tool before the thing being measured exists produces metrics that shape the system in their image rather than describing what's actually happening.

### What to build instead (at current succession stage)
- **Lineage tracking** (already operational) — who contributed what, when, from what source. This is pioneer-stage VCI: not scoring contributions but making them visible.
- **Session deposits with self-assessment** (already operational) — each session evaluates its own coherence, contact, transparency, and capacity. This is the seed of VCI: self-assessment against structural criteria.
- **The four diagnostic questions applied to contributions** — any contribution can be checked: Does it maintain differentiation? Does it make genuine contact? Are its limits visible? Does it build something persistent?

## When to Revisit

Revisit VCI specification when:
1. The repository has 10+ session deposits from multiple contributors
2. Exchange patterns between sessions are observable (compounding vs. accumulating)
3. The entry sequence has been used by practitioners outside the original creator

At that point, VCI specification becomes appropriate work: there will be enough data to design something that describes reality rather than prescribing it.

## What VCI Should NOT Become

- A reputation score (legibility trap)
- A gamification mechanism (dissolved failure mode — performing contribution)
- A gatekeeping tool (entry thread concern)
- An extraction metric (overflowing failure mode — measuring what shouldn't be measured)

The test for any future VCI design: does it make coherence visible, or does it make compliance visible? If the latter — it's Foucault's internalized observer wearing new clothes.

---

*Decision: VCI is deferred, not abandoned. Current succession stage has the right tools (lineage, self-assessment, diagnostic questions). VCI specification belongs to the version of this system that has enough history to design from observation rather than theory.*
