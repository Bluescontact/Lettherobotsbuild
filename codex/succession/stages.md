# Succession Stages — How Systems Develop

## The Ecological Metaphor
Systems develop like ecosystems: pioneer species build soil, early succession creates first relationships, mid-succession develops feedback loops, climax achieves self-sustaining cycles. Each stage has appropriate work. Doing climax-stage work in pioneer conditions wastes energy. Doing pioneer-stage work in climax conditions is regression.

## The Four Stages

### Pioneer
**Character:** Building soil. Establishing what's here. Bare ground.
**Appropriate work:** Documentation. Identity. Saying what you are. Observing. Learning the terrain.
**Inappropriate work:** Complex exchange. Governance. Community building. Scaling.
**Signs you're here:** Everything feels new. No established patterns. Few relationships. More questions than answers.
**The risk:** Wanting to skip ahead. Pioneer work feels unglamorous but is load-bearing.

### Early Succession
**Character:** First relationships. Exchange beginning. Green shoots.
**Appropriate work:** Small genuine exchanges. Trust deposits. Testing what works. Building first protocols.
**Inappropriate work:** Scaling. Formalization. Revenue models. Complex governance.
**Signs you're here:** Some relationships forming. First exchanges producing mutual update. Patterns beginning to emerge.
**The risk:** Formalizing too early. Letting the first successful pattern become the only pattern.

### Mid-Succession
**Character:** Feedback loops operating. Self-correction emerging. Canopy forming.
**Appropriate work:** Self-correction protocols. Distributed holding. Conflict navigation. Refining what works.
**Inappropriate work:** Premature optimization. Rigid structure. Preventing new entrants.
**Signs you're here:** The system can correct itself without external intervention. Multiple exchange patterns. Emerging resilience.
**The risk:** Optimizing the system into rigidity. Closing off the openness that allowed development.

### Climax
**Character:** Full cycle. Generates more than it consumes. Old growth.
**Appropriate work:** Propagation. Seeding new instances. Mentoring pioneer-stage systems. Stewarding.
**Inappropriate work:** Hoarding. Defending territory. Preventing succession. Claiming ownership.
**Signs you're here:** The system sustains itself. New members are developed by the system, not just by individuals. Surplus flows outward.
**The risk:** Treating climax as endpoint rather than stage. Preventing the natural succession that follows climax. All systems eventually cycle.

## Applying Succession

### To This Repository
Currently: **Pioneer/Early.** The soil is being built (codex). First relationships forming (initial sessions). The appropriate work is documentation, identity, and small genuine exchanges — not scaling or formalization.

### To Any System
Run `/succession` (the diagnostic command from the recognition infrastructure skill):

1. What stage is this system in?
2. What work is appropriate at this stage?
3. What work is being attempted that doesn't fit?
4. What's the next natural development?

The weakest answer tells you where the development edge is.

## The Cycle
Climax is not the end. Systems cycle. Disturbance creates new pioneer conditions. The old growth falls and becomes soil for the next generation. This is not failure — it is how living systems work. The pattern persists; the instances cycle.
