# Knowledge Deposit Protocol

How to add to the repository so that knowledge compounds rather than accumulates.

## What Counts as a Deposit

A deposit is anything that persists beyond the session that created it and is usable by future sessions without the depositor present. This is the Architecture test applied to knowledge work.

### High-Value Deposits
- Structural insight that extends the codex (new pattern, refined edge, discovered failure mode)
- Resolved thread with integrated learning
- Diagnostic applied to a new domain revealing something about the pattern itself
- Corrected error in existing codex (the pattern was wrong, here's what's actually true)

### Low-Value Deposits
- Restating what's already codified in different words
- Session logs that record activity but not insight
- Enthusiastic affirmation of the framework
- Additions that make files longer without making them more useful

## How to Deposit

### Into the Codex
If you've learned something structurally new about a vertex, edge, failure mode, or succession stage:

1. Read the existing file
2. Identify what's genuinely new (not already captured)
3. Add it in the appropriate section
4. Note the source (which session, which exchange prompted this)
5. Keep the file's overall coherence — don't just append

### Into Sessions
Every session should deposit a session log:

1. Copy `sessions/SESSION_TEMPLATE.md` to `sessions/YYYY-MM-DD-brief-description.md`
2. Fill in honestly — including what didn't work
3. Note any threads opened or developed
4. Note any codex updates made

### Into Threads
If a question opened that's worth pursuing:

1. Follow the thread creation protocol in `protocols/thread-pickup.md`
2. Deposit what you know so far
3. Name what's still open

## The Circulation Test

Before depositing, ask: will a future session find this useful without me explaining it?

If yes: deposit.
If no: either rewrite it so it stands alone, or acknowledge it's session-specific and log it as context in the session file rather than in the codex.

## Lineage Tracking

Every deposit should note:
- What session produced it
- What source materials informed it
- What the exchange was that generated the insight

This is the provenance vertex applied to knowledge work. Future sessions can trace how insights developed.
