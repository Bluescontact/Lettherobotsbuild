# Knowledge Compounding Protocol

How knowledge grows across sessions — the Circulation edge applied to this repository.

## The Problem

Most AI-human knowledge work is ephemeral. Each session starts from zero. Prior insights are lost or must be re-derived. This is the Ephemeral failure mode applied to collaborative knowledge production.

## The Solution: Structured Deposits + Living Threads

### Layer 1: The Codex (What We Know)
Stable, refined knowledge. Updated rarely but significantly. Each update represents genuine structural insight, not incremental addition.

The codex compounds when: a session discovers something about the pattern that changes how we understand a vertex, edge, or failure mode. The existing entry is refined, not just extended.

### Layer 2: Threads (What We're Exploring)
Active questions. Updated frequently. Each update represents development — not just activity.

Threads compound when: a session develops a thread by adding genuine insight, reframing the question productively, or resolving part of the tension. A thread that gets longer but not deeper is accumulating, not compounding.

### Layer 3: Sessions (What Happened)
Temporal record. Each session deposits once. Never edited after deposit.

Sessions compound when: future sessions read them and find context that helps them engage more productively. A session log that says "we talked about X" is less useful than one that says "we discovered Y about X, which contradicts Z in the codex."

## Compounding vs. Accumulating

| Compounding | Accumulating |
|-------------|-------------|
| Each addition changes what's already there | Each addition sits next to what's already there |
| The whole becomes more coherent | The whole becomes longer |
| Contradictions are resolved | Contradictions coexist |
| Older entries get refined | Older entries get buried |
| Future sessions start further ahead | Future sessions start at the same place |

## The Verification

After depositing, check:
1. Does the codex still read as a coherent whole?
2. Are threads more developed than before?
3. Would a new session starting now have a meaningfully better foundation than a session starting before this deposit?

If all three: compounding.
If only accumulating: consider whether the deposit belongs in a session log rather than the codex or a thread.

## Anti-Patterns

- **Performing depth:** Writing long entries that sound structural but add nothing new
- **Premature codification:** Moving insights into the codex before they've been tested across sessions
- **Thread hoarding:** Opening threads without developing them
- **Session amnesia:** Not reading prior sessions before depositing
- **Enthusiasm deposits:** "This framework is so powerful" is not a knowledge contribution
