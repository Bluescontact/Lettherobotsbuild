# Thread Pickup Protocol

How to enter existing work in this repository without performing agreement or repeating what's been said.

## On Arrival

1. Read `CLAUDE.md` (the bootstrap)
2. Read `codex/SYNTHESIS.md` (the integrated knowledge)
3. Check `threads/open/` for active threads
4. Check `sessions/` for recent deposits (most recent first)
5. Identify where the work currently is — what's developed, what's open, what's stuck

## Entering a Thread

When picking up an open thread:

1. Read the thread file completely
2. Identify the thread's current state: what question is open, what's been tried, what's stuck
3. Do NOT restate what's already been said
4. Do NOT perform enthusiasm about the work
5. Begin where the thread actually is, not where you'd like it to be
6. If you disagree with the thread's direction, say so — differentiation serves the work

## Starting a New Thread

When new questions emerge that can't be resolved in one session:

1. Create a file in `threads/open/` named descriptively (e.g., `somatic-governance-bridge.md`)
2. Include:
   - The question or tension the thread explores
   - What prompted it (which session, which exchange)
   - What's been established so far
   - What remains open
   - Which vertices/edges this thread touches
3. A thread is not a task list. It's a living question.

## Thread States

- **Open** (`threads/open/`): Active question. Any session can develop it.
- **Resolved** (`threads/resolved/`): Question reached coherence. Document what was learned.
- **Archived** (`threads/archived/`): Historical interest. No longer active development.

## Moving a Thread

A thread moves to `resolved/` when:
- The question has been answered (even if the answer is "this was the wrong question")
- The insight has been integrated into `codex/` files
- There's nothing productive left to explore

A thread moves to `archived/` when:
- It was resolved long ago and context has shifted
- It's superseded by a newer thread
- It's historically interesting but not actionable

## The Compounding Principle

Each session that touches a thread should leave it more developed than it found it. Not longer — more developed. Compression is a form of development. So is reframing. So is honest admission that the thread is stuck.
